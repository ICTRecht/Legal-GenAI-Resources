# Skill Template

Use this template when creating a new Skill. Copy the folder structure below and replace the placeholder content.

---

## Folder structure

```
skills/{skill-name}/
├── skill.md          # The system prompt — paste this as the system prompt in Claude
├── README.md         # Purpose, legal context, usage, limitations
├── example_output.md # A real or illustrative example of what this Skill produces
└── metadata.json     # Version, validation, and regulatory metadata
```

---

## skill.md — template

```markdown
# {Skill Name}

## Role
You are {role description}. You operate under {applicable law/jurisdiction}.

## Behaviour
- {Instruction 1}
- {Instruction 2}
- {Instruction 3}

## Output format
{Describe the expected output structure — sections, headers, tables, etc.}

## Limitations
Always include the following disclaimer in your output:
> This output is generated by an AI assistant and does not constitute legal advice. Verify all findings with a qualified lawyer before relying on them.

## Scope
- In scope: {what this Skill covers}
- Out of scope: {what this Skill does NOT cover}
```

---

## README.md — template

```markdown
# {Skill Name}

**Status:** draft | reviewed | production
**ICTRecht Verified:** yes | no

## What this Skill does
{1-2 sentences describing the Skill's purpose and the legal task it supports.}

## Applicable law
{Which regulation or body of law does this Skill operate under? E.g. GDPR (Regulation (EU) 2016/679), EU AI Act (Regulation (EU) 2024/1689), Dutch contract law.}

## Jurisdiction
{E.g. Netherlands, European Union, international}

## How to use
1. Open `skill.md` and copy the full contents.
2. Paste it as a system prompt in Claude (Project Instructions, or the system prompt field in the API).
3. Start your session — Claude now behaves as the configured specialist.

## Compatible models
{E.g. Claude claude-opus-4-7, Claude claude-sonnet-4-6, GPT-4o}

## Known limitations
- {Limitation 1}
- {Limitation 2}
```

## metadata.json — template

```json
{
  "model": "claude-opus-4-7",
  "validated_by": "ICTRecht",
  "validation_date": "YYYY-MM-DD",
  "regulation_version": "e.g. GDPR as of 2024-05",
  "status": "draft"
}
```
